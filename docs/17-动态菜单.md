# 17 - 动态菜单（RBAC权限）

> 基于八步法分析 yudao-cloud 中的动态菜单实现

---

## ① Why - 价值（为什么）

### 背景与痛点

在传统权限系统中，菜单通常是**静态配置**的：

| 痛点 | 说明 |
|------|------|
| **修改困难** | 菜单变更需要修改代码，重新部署 |
| **权限控制粗** | 无法精细到按钮级别 |
| **无法个性化** | 所有用户看到相同菜单 |
| **多租户不便** | 不同租户需要不同菜单 |

### 动态菜单价值

| 价值 | 说明 |
|------|------|
| **灵活配置** | 通过后台管理界面动态配置菜单 |
| **精细权限** | 支持目录/菜单/按钮三级权限 |
| **角色关联** | 菜单与角色关联，按角色展示 |
| **多租户隔离** | 不同租户拥有不同菜单 |

### 适用场景

- 企业管理系统（不同角色看到不同功能）
- SaaS 系统（不同租户需要不同菜单）
- 开放平台（不同客户等级看到不同 API）

## ② What - 定义（是什么）

### 一句话定义

**动态菜单**是一种基于角色的访问控制（RBAC）机制，菜单数据存储在数据库中，用户登录时根据角色动态生成菜单树。

### 核心概念

| 概念 | 说明 |
|------|------|
| **菜单类型** | 目录（Dir）、菜单（Menu）、按钮（Button） |
| **菜单树** | 父子层级结构组成的菜单树 |
| **权限标识** | 唯一标识菜单的权限字符 |
| **组件** | 前端 Vue 组件路径 |

### yudao-cloud 菜单类型

```
┌─────────────────────────────────────────────────┐
│              菜单层级结构                        │
├─────────────────────────────────────────────────┤
│  目录 (Dir) - 侧边栏一级目录                    │
│    ├── 子目录 (Dir) - 可嵌套                    │
│    └── 菜单 (Menu) - 具体页面                   │
│          └── 按钮 (Button) - 操作权限           │
└─────────────────────────────────────────────────┘
```

#### 菜单树构建

```java
public List<AuthMenuRespVO> buildMenuTree(List<MenuDO> menus) {
    // 1. 按 parent_id 分组
    Map<Long, List<MenuDO>> parentMap = menus.stream()
        .collect(Collectors.groupingBy(MenuDO::getParentId));
    
    // 2. 递归构建树
    return buildTree(0L, parentMap);
}

private List<AuthMenuRespVO> buildTree(Long parentId, 
        Map<Long, List<MenuDO>> parentMap) {
    List<AuthMenuRespVO> result = new ArrayList<>();
    List<MenuDO> children = parentMap.get(parentId);
    
    if (CollUtil.isEmpty(children)) {
        return result;
    }
    
    for (MenuDO menu : children) {
        AuthMenuRespVO vo = BeanUtils.toBean(menu, AuthMenuRespVO.class);
        vo.setChildren(buildTree(menu.getId(), parentMap));
        result.add(vo);
    }
    
    // 3. 排序
    result.sort(Comparator.comparing(AuthMenuRespVO::getSort));
    return result;
}
```

## ④ Hard - 难点（挑战）

### 难点1：菜单层级过深

**问题**：菜单层级过多导致前端渲染复杂

**解决方案**：限制菜单层级（建议最多3层）

```java
// 菜单创建时校验层级
private void validateMenuLevel(Long parentId) {
    int level = 1;
    Long currentId = parentId;
    while (currentId != null && currentId != 0L) {
        MenuDO parent = menuMapper.selectById(currentId);
        if (parent == null) break;
        level++;
        currentId = parent.getParentId();
    }
    if (level > 3) {
        throw exception(MENU_LEVEL_EXCEEDED);
    }
}
```

### 难点3：按钮权限

**问题**：如何控制按钮级别的权限？

### 难点4：多租户菜单隔离

**问题**：不同租户需要不同菜单

```java
@Service
public class MenuServiceImpl implements MenuService {
    
    @Resource
    private MenuMapper menuMapper;
    
    @Override
    public List<MenuDO> getMenuListByTenant(MenuListReqVO reqVO) {
        // 1. 获取租户菜单
        List<MenuDO> menus = getMenuList(reqVO);
        
        // 2. 过滤禁用的菜单
        return filterDisableMenus(menus);
    }
    
    @Override
    public List<MenuDO> filterDisableMenus(List<MenuDO> menuList) {
        // 过滤 status = 1 的菜单
        return menuList.stream()
            .filter(menu -> menu.getStatus().equals(CommonStatusEnum.ENABLE.getCode()))
            .collect(Collectors.toList());
    }
}
```

### 脚本

```sql
-- 查询所有菜单（树形）
WITH RECURSIVE menu_tree AS (
  SELECT id, parent_id, name, path, icon, type, 0 AS level
  FROM system_menu
  WHERE parent_id = 0
  
  UNION ALL
  
  SELECT m.id, m.parent_id, m.name, m.path, m.icon, m.type, mt.level + 1
  FROM system_menu m
  JOIN menu_tree mt ON m.parent_id = mt.id
)
SELECT * FROM menu_tree ORDER BY level, sort;
```
