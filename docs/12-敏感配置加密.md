# 12-敏感配置加密.md

> 配置管理 - 敏感配置加密

---

## ① Why - 价值 (为什么)

**背景与痛点**

- **背景**：在分布式系统中，存在大量敏感配置（数据库密码、API密钥、Token等），直接存储有泄露风险
- **不用它的痛点**：
  - 配置文件明文存储敏感信息，被攻击者拖库后直接泄露
  - 敏感数据在日志中暴露（如打印用户手机号、身份证）
  - 数据库字段明文存储（如密码、银行卡号）
  - API请求/响应中的敏感信息被截获
- **收益**：
  - 敏感配置加密存储，即使被拖库也无法直接使用
  - API请求/响应加密，防止网络层截获
  - 数据库敏感字段加密，满足合规要求（如PCI-DSS）
  - 统一加解密框架，便于管理和审计

**目标用户**

- 运维人员：密钥管理和轮换
- 开发人员：敏感数据标识和处理
- 安全审计：加密合规检查

---

## ② What - 定义 (是什么)

**一句话定义**

敏感配置加密是指对系统中的敏感信息（密码、密钥、Token、个人隐私等）进行加密存储和传输，防止信息泄露的安全保护机制。

**核心概念**

| 概念 | 说明 |
|------|------|
| 对称加密 | 加密解密用同一密钥（如AES、SM4），适合数据量大的场景 |
| 非对称加密 | 公钥加密私钥解密（如RSA、SM2），适合密钥交换 |
| 字段级加密 | 数据库字段存储时加密（如手机号、身份证） |
| API加解密 | HTTP请求/响应 Body 加密传输 |
| 密钥管理 | 加密密钥的存储、轮换、审计 |

**核心流程**

```
明文数据 → 加密 → 存储/传输 → 使用时解密 → 明文数据
```

## ④ Hard - 难点 (挑战)

### 可能遇到的问题

**难点1：密钥如何安全存储？**
```
场景：加密密钥存在配置文件中，配置文件被拖库等于明文泄露
→ 解决方案：密钥存储在外部（如KMS、Vault），运行时动态获取
```

**难点2：加密后如何检索？**
```
场景：手机号加密后无法模糊查询
→ 解决方案：保留明文索引字段，或使用可搜索加密（确定性加密）
```

**难点3：密钥轮换如何平滑过渡？**
```
场景：更换加密密钥，历史数据如何解密
→ 解决方案：支持双密钥，渐进式迁移
```

**难点4：性能影响**
```
场景：大量数据加解密导致接口响应变慢
→ 解决方案：异步加解密、缓存热点数据、硬件加速
```

### 边界情况

- 字段为NULL时的加密处理
- 加密算法升级时的历史数据兼容
- 多租户场景下的密钥隔离

### 候选方案对比

| 方案 | 优点 | 缺点 | 适用场景 |

| 自研加解密框架 | 完全可控 | 投入开发量大 | 有安全团队的公司 |

### 选型理由

**芋道选择方案：AES + Hutool**

1. 依赖轻量级加密库（Hutool），无需额外部署
2. 提供两种模式：API加解密、字段级加密
3. 与Spring生态深度整合
4. 配置灵活，可按需开启

## ⑦ Impl - 实现 (细节)

### 数据模型

```java
// 配置实体 - 敏感配置示例
public class SensitiveConfigDO {
    private Long id;
    private String name;
    private String configKey;
    private String configValue;      // 存储密文
    private String algorithm;        // AES/RSA
    private String remark;
}

// 用户实体 - 密码字段加密示例
@TableName("system_user")
public class AdminUserDO {
    private Long id;
    private String username;
    
    // 密码字段，使用@Encrypt注解自动加解密
    @TableField(typeHandler = EncryptTypeHandler.class)
    private String password;
    
    private String mobile;
    private String email;
}
```

### 核心代码

**1. API加解密配置**

```java
// application.yml 配置
yudao:
  api-encrypt:
    enable: true
    header: X-Api-Encrypt
    algorithm: AES
    requestKey: your-request-key       # 后端解密密钥
    responseKey: your-response-key      # 后端加密密钥
```

**2. API加解密过滤器**

```java
// ApiEncryptFilter.java - 核心逻辑
public class ApiEncryptFilter extends ApiRequestFilter {

    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                    HttpServletResponse response, 
                                    FilterChain chain) {
        // 1. 获取 @ApiEncrypt 注解
        ApiEncrypt apiEncrypt = getApiEncrypt(request);
        
        // 2. 解密请求（POST/PUT/DELETE）
        if (需要解密) {
            request = new ApiDecryptRequestWrapper(request, 
                symmetricDecryptor, asymmetricDecryptor);
        }
        
        // 3. 执行后续过滤器
        chain.doFilter(request, response);
        
        // 4. 加密响应
        if (需要加密响应) {
            response.encrypt(responseEncryptor);
        }
    }
}
```

**3. 使用注解**

```java
// 在Controller上使用
@RestController
@RequestMapping("/user")
@ApiEncrypt  // 整个Controller加解密
public class UserController {
    
    @PostMapping("/login")
    @ApiEncrypt(request = true, response = true)  // 单独方法
    public CommonResult<LoginResp> login(@RequestBody LoginReq req) {
        // ...
    }
}
```

**4. 数据库字段加密**

```java
// application.yml 配置密钥
mybatis-plus:
  encryptor:
    password: your-encrypt-key

// 实体类使用
@TableName("system_user")
public class AdminUserDO {
    
    // 自动加解密
    @TableField(typeHandler = EncryptTypeHandler.class)
    private String password;
    
    @TableField(typeHandler = EncryptTypeHandler.class)
    private String idCard;
    
    @TableField(typeHandler = EncryptTypeHandler.class)
    private String bankCard;
}
```

**5. 加密TypeHandler实现**

```java
// EncryptTypeHandler.java
public class EncryptTypeHandler extends BaseTypeHandler<String> {

    private static AES aes;

    // 写入时加密
    @Override
    public void setNonNullParameter(PreparedStatement ps, int i, 
                                    String parameter, JdbcType jdbcType) {
        ps.setString(i, encrypt(parameter));
    }

    // 读取时解密
    @Override
    public String getNullableResult(ResultSet rs, String columnName) {
        String value = rs.getString(columnName);
        return decrypt(value);
    }

    private static String encrypt(String rawValue) {
        if (rawValue == null) return null;
        return getEncryptor().encryptBase64(rawValue);
    }

    private static String decrypt(String value) {
        if (value == null) return null;
        return getEncryptor().decryptStr(value);
    }
}
```

### 关键步骤校验

```
Step 1: 配置密钥
  → 校验点：密钥已配置且不为空
  → 验证方法：检查application.yml

Step 2: 开启API加解密
  → 校验点：yudao.api-encrypt.enable=true
  → 验证方法：请求带header测试

Step 3: 数据库字段加密
  → 校验点：字段使用EncryptTypeHandler
  → 验证方法：查询数据库确认密文存储

Step 4: 性能测试
  → 校验点：加解密耗时<50ms
  → 验证方法：压测报告
```

### 约束条件

- 不修改已有API接口签名
- 加解密耗时<50ms（单次）
- 密钥长度建议16位以上（AES）
- 兼容原有明文数据（支持平滑迁移）

```
场景1：需要保护API请求/响应中的敏感数据

场景3：需要满足安全合规（如等保、PCI-DSS）
```

### 执行流程

```
Step 1: 引入依赖
  - Hutool Crypto模块（已集成）

Step 2: 配置加解密密钥
  - application.yml 配置 mybatis-plus.encryptor.password
  - 或 yudao.api-encrypt.requestKey/responseKey

Step 3: API加解密
  - 启用 yudao.api-encrypt.enable=true
  - Controller或方法上使用 @ApiEncrypt 注解

Step 4: 字段级加密
  - 实体类字段使用 @TableField(typeHandler = EncryptTypeHandler.class)
  - MyBatis自动处理加解密
```
