# 02-执行器接入

> 定时任务 - 执行器接入与配置详解

---

## ① Why - 价值

**背景与痛点**：
- 执行器需要手动编码启动，配置繁琐
- 每加一个服务都要重复配置调度中心地址
- 多环境（dev/test/prod）配置切换麻烦

**收益**：
- 只需配置一次，所有服务自动接入
- Spring Boot 自动装配，零代码接入
- 多环境配置管理更简单

**用户**：
- 后端开发者、运维工程师

---

## ② What - 定义

**一句话定义**：
执行器是 XXL-Job 的客户端，负责接收调度中心的指令并执行任务。

**核心组成**：
1. **XxlJobExecutor** - 执行器核心类，负责与调度中心通信
2. **XxlJobSpringExecutor** - Spring 环境下使用的执行器实现
3. **XxlJobProperties** - 配置属性类

**关键术语**：
- appName - 执行器应用名，用于调度中心识别
- admin.addresses - 调度中心地址列表
- executor.port - 执行器监听端口（默认随机）

---

## ③ How - 思维

**目标**：实现执行器自动接入

**范围**：
- 允许：`yudao-framework/yudao-spring-boot-starter-job/`
- 禁止：修改业务模块的 Service 层

**禁止**：
- 禁止修改已有执行器配置类
- 禁止改动调度中心通信协议

**配置模型**：

```yaml
xxl:
  job:
    enabled: true  # 是否开启
    admin:
      addresses: http://127.0.0.1:9090/xxl-job-admin  # 调度中心地址
    executor:
      appName: yudao-infra  # 应用名
      ip: 192.168.1.100    # 执行器IP（可选，自动检测）
      port: -1             # 端口（-1表示随机）
      logPath: /tmp/xxl-job  # 日志路径
      logRetentionDays: 30  # 日志保留天数
    accessToken: xxx        # 访问令牌（安全用）
```

**核心流程**：
```
应用启动 
  → YudaoXxlJobAutoConfiguration 
  → 初始化 XxlJobSpringExecutor 
  → 注册到调度中心 
  → 监听调度指令 
  → 执行任务
```

**关键类**：
- `YudaoXxlJobAutoConfiguration` - 自动配置入口
- `XxlJobProperties` - 配置属性（对应 xxl.job.*）
- `XxlJobExecutor` - 执行器底层实现

## ④ Hard - 难点

**难点1：多环境配置切换？**
```
场景：dev/test/prod 环境调度中心地址不同
→ 使用 Spring Profile 或 application-{env}.yaml
```

**难点2：执行器注册失败？**
```
场景：调度中心没启动或网络不通
→ 启动检查日志，确认 admin.addresses 正确
```

**难点3：端口冲突？**
```
场景：多个服务部署在同一台机器，port 相同
→ 设置 port: -1 使用随机端口
```

## ⑥ Select - 选型

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| Spring Boot Starter | 自动装配、配置简单 | 依赖框架 | yudao-cloud 项目 |
| 手动 new XxlJobExecutor | 完全控制 | 代码繁琐 | 自定义需求 |
| 注解 @EnableXxlJob | 显式启用 | 多一步操作 | 明确启用场景 |

**选型理由**：
1. yudao-spring-boot-starter-job 已封装好，开箱即用
2. 配置式接入，符合 Spring Boot 最佳实践
3. 支持多环境配置管理

## ⑦ Impl - 实现

**配置属性类**：

```java
@ConfigurationProperties("xxl.job")
public class XxlJobProperties {
    private Boolean enabled = true;
    private String accessToken;
    private AdminProperties admin;
    private ExecutorProperties executor;
    
    @Data
    public static class AdminProperties {
        private String addresses;
    }
    
    @Data
    public static class ExecutorProperties {
        private String appName;
        private String ip;
        private Integer port = -1;  // 随机端口
        private String logPath;
        private Integer logRetentionDays = 30;
    }
}
```

**自动配置类**：

```java
@AutoConfiguration
@ConditionalOnProperty(prefix = "xxl.job", name = "enabled", havingValue = "true")
@EnableConfigurationProperties({XxlJobProperties.class})
public class YudaoXxlJobAutoConfiguration {
    
    @Bean
    @ConditionalOnMissingBean
    public XxlJobExecutor xxlJobExecutor(XxlJobProperties properties) {
        XxlJobExecutor executor = new XxlJobSpringExecutor();
        executor.setAppname(properties.getExecutor().getAppName());
        executor.setAdminAddresses(properties.getAdmin().getAddresses());
        executor.setPort(properties.getExecutor().getPort());
        executor.setLogPath(properties.getExecutor().getLogPath());
        // ... 其他配置
        return executor;
    }
}
```

**使用示例**：

```yaml
# application-local.yaml
xxl:
  job:
    enabled: true
    admin:
      addresses: http://127.0.0.1:9090/xxl-job-admin
    executor:
      appName: yudao-infra
      logPath: /tmp/xxl-job
```

**关键步骤校验**：
- Step 1: 检查 enabled=true
- Step 2: 校验 admin.addresses 不为空
- Step 3: 校验 executor.appName 不为空
- Step 4: 启动时自动注册

## ⑧ SKILL - 提炼

**触发条件**：
- 场景1：新服务需要接入 XXL-Job
- 场景2：切换 XXL-Job 调度中心地址
- 场景3：配置多环境执行器

**执行流程**：
```
Step 1: 引入依赖
  - yudao-spring-boot-starter-job 已包含 xxl-job-core

Step 2: 配置 application.yml
  - xxl.job.enabled: true
  - xxl.job.admin.addresses
  - xxl.job.executor.appName

Step 3: 启动服务
  - 自动注册到调度中心
  - 调度中心查看执行器在线
```

**配方**：
- 依赖：xxl-job-core (通过 starter 引入)
- 配置：xxl.job.* (application.yml)
- 端口：默认随机（-1）

**验收标准**：
- [ ] 服务启动日志显示执行器初始化成功
- [ ] 调度中心能看到执行器在线
- [ ] 能正常接收调度指令
