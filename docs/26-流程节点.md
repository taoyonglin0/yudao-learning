## ① Why - 价值 (为什么)

### 背景与痛点

在 BPMN 流程设计中，流程节点是核心组成部分：

1. **节点类型单一**：只能做简单审批，无法满足复杂业务场景
2. **连线条件复杂**：条件表达式的配置和维护困难
3. **节点行为固化**：审批、会签、转交等行为无法灵活配置
4. **边界事件缺失**：超时处理、错误处理等边界事件支持不足

### 收益

- **多类型节点**：支持用户任务、服务任务、脚本任务等多种节点
- **灵活连线**：支持排他、并行、包含网关及条件表达式
- **可配置行为**：每个节点的行为可单独配置
- **边界事件**：支持超时、错误等边界事件处理

### 使用者

- 流程设计人员
- 后端开发工程师
- 业务分析师

## ② What - 定义 (是什么)

### 一句话定义

**流程节点与连线**是指 BPMN 流程中的人工事务节点、自动执行节点、网关节点及其连接线的配置与管理。

### 核心组成部分

```
┌─────────────────────────────────────────────────────────────┐
│                    BPMN 流程节点                         │
├──────────┬──────────┬──────────┬──────────┬────────────┤
│ 用户任务 │ 服务任务 │ 脚本任务 │ 网关节点 │  边界事件  │
│(UserTask)│(Service)│(Script) │(Gateway)│(Boundary) │
├──────────┴──────────┴──────────┴──────────┴────────────┤
│                    连线 (SequenceFlow)                  │
│         条件表达式 │ 排他网关 │ 并行网关                │
└─────────────────────────────────────────────────────────────┘
```

### 关键术语

| 术语 | 说明 |
|------|------|
| UserTask | 用户任务，需要人工处理 |
| ServiceTask | 服务任务，自动执行 |
| ScriptTask | 脚本任务，脚本执行 |
| ExclusiveGateway | 排他网关，满足条件即出口 |
| ParallelGateway | 并行网关，全部通过 |
| InclusiveGateway | 包含网关，条件通过即出口 |
| SequenceFlow | 顺序流，连接节点 |

## ④ Hard - 难点 (挑战)

### 难点 1：多实例任务（会签/或签）

**场景**：一个任务需要多个人审批，如何实现会签（全部同意）或签（任一同意）。

**解决方案**：Flowable MultiInstance
- Sequential：顺序执行
- Parallel：并行执行
- Cardinality：循环次数
- LoopCardinality：循环基数

### 难点 2：审批人 assignee 动态分配

**场景**：审批人需要根据业务变量动态确定（如部门负责人）。

**解决方案**：实现 TaskCandidateStrategy
- 用户策略
- 角色策略
- 部门负责人策略
- 表达式策略

### 难点 3：节点超时与边界事件

**场景**：审批超时未处理，需要自动催办或自动通过。

**解决方案**：边界事件 + 定时器
- TimerBoundaryEvent
- 触发条件：duration: PT1H
- 处理方式：发送通知 / 自动通过 / 自动拒绝

### 难点 4：服务任务调用外部系统

**场景**：流程节点需要调用外部 API。

**解决方案**：ServiceTask + Delegate
- JavaDelegate
- Expression
- HTTP Connector

### 选型理由

**选择 Flowable**，因为：
1. 完整支持 BPMN 2.0 所有节点类型
2. 多实例支持完善（会签/或签）
3. 边界事件功能完整
4. 与 Spring Boot 无缝集成

### 失败恢复机制

- **场景 1：审批人不可用**
  - 触发：assignee 离职
  - 恢复：降级到候选组

- **场景 2：服务任务失败**
  - 触发：外部调用超时
  - 恢复：重试机制 + 补偿

- **场景 3：超时边界事件**
  - 触发：审批超时
  - 恢复：发送通知
