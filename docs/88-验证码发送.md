# 88-验证码发送.md

> 验证码 - 验证码发送

---

## ① Why - 价值 (为什么)

**背景与痛点**

- **背景**：用户登录、注册、修改密码等敏感操作需要身份验证，防止恶意批量注册、暴力破解密码
- **不用它的痛点**：
  - 用户密码容易被暴力枚举
  - 短信/邮箱被恶意调用造成资源浪费
  - 无法区分真人还是机器攻击
- **收益**：
  - 防止自动化攻击，提升安全性
  - 减少短信/邮箱验证码被刷，节约成本
  - 用户体验更好（相对于传统滑块更安全）

**目标用户**

- 需要身份验证的业务场景（登录、注册、找回密码、支付等）
- 管​理后台和用户端均需要验证码

---

## ② What - 定义 (是什么)

**一句话定义**

验证码是一种人机识别技术，通过识别用户行为或答案校验来确认操作者是真人而非机器/自动化脚本。

**核心组成**

1. **验证码类型**
   - 滑动拼图（drag）
   - 文字点选（click）
   - 文字识图（clickWord）
   - 短信/邮箱验证码（code）

2. **核心流程**
   - 生成验证码 → 返回前端展示 → 用户验证 → 后端校验 → 返回结果

3. **关键术语**
   - `CaptchaVO`：验证码传输对象
   - `CaptchaService`：验证码服务
   - `CaptchaCacheService`：验证码缓存服务
   - `AjCaptcha`：集成验证码框架

## ③ How - 思维 (怎么做)

### 数据模型设计

```
验证码配置表（aj_captcha）：
- id               : 主键
- captcha_type     : 验证码类型
- cache_type       : 缓存类型（redis/memory）
- token            : 验证码唯一标识
- verify           : 校验值
- create_time      : 创建时间
- expires_in       : 过期时间（秒）

Redis 存储结构：
- captcha:token:{token}     : 验证码答案（set）
- captcha:verify:{token}    : 验证状态（set after check）
```

### 关键流程设计

```
前端请求 → 生成验证码（CaptchaService.get） → 返回验证码图片/token → 
用户滑动/点选/输入 → 提交验证 → 
后端校验（CaptchaService.check） → 返回验证结果
```

### 代码设计

**关键类**

| 类名 | 职责 |
|------|------|
| `CaptchaController` | 验证码HTTP接口 |
| `CaptchaService` | 验证码核心服务 |
| `CaptchaCacheService` | 验证码缓存抽象 |
| `RedisCaptchaServiceImpl` | Redis缓存实现 |
| `YudaoCaptchaConfiguration` | 验证码配置类 |

## ④ Hard - 难点 (挑战)

### 问题1：验证码重复使用

```
场景：同一个token可以多次验证，第一-次成功后仍可使用
→ 解决：验证成功后删除token，确保一次性
```

### 问题2：缓存不一致

```
场景：多节点部署时，验证码存在本地内存，其他节点无法访问
→ 解决：使用Redis集中式缓存（yudao已实现）
```

### 问题3：Spring Boot 3.x 兼容

```
场景：aj-captcha对Spring Boot 3.x自动配置不生效
→ 解决：手动配置@ImportAutoConfiguration
```

### 问题4：IP伪造

```
场景：攻击者伪造IP绕过频率限制
→ 解决：结合反爬虫、频率限制、行为分析
```

## ⑥ Select - 选型 (选哪个)

### 候选方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| AJ-Captcha | 前后端分离、类型丰富、国产 | 文档较少 | 国内项目 |
| geetest | 成熟稳定、市场占有率高 | 收费、商业化 | 企业项目 |
| hCaptcha | 国际通用、隐私友好 | 国内访问慢 | 国际化项目 |
| Turnstile | Cloudflare背书、隐蔽性好 | 依赖CDN | 高安全场景 |

### 选型结论

**选 AJ-Captcha**，因为：
1. 国产开源，文档中文
2. 与芋道技术栈匹配（Spring Boot）
3. 支持Redis集中式缓存，解决分布式问题
4. 前后端分离架构，前端轻量

## ⑦ Impl - 实现 (细节)

### 核心代码

**验证码Controller**

```java
@Tag(name = "管理后台 - 验证码")
@RestController("adminCaptchaController")
@RequestMapping("/system/captcha")
public class CaptchaController {

    @Resource
    private CaptchaService captchaService;

    @PostMapping({"/get"})
    @Operation(summary = "获得验证码")
    @PermitAll
    @TenantIgnore
    public ResponseModel get(@RequestBody CaptchaVO data, HttpServletRequest request) {
        assert request.getRemoteHost() != null;
        data.setBrowserInfo(getRemoteId(request));
        return captchaService.get(data);
    }

    @PostMapping("/check")
    @Operation(summary = "校验验证码")
    @PermitAll
    @TenantIgnore
    public ResponseModel check(@RequestBody CaptchaVO data, HttpServletRequest request) {
        data.setBrowserInfo(getRemoteId(request));
        return captchaService.check(data);
    }
}
```

**Redis缓存实现**

```java
@Setter
public class RedisCaptchaServiceImpl implements CaptchaCacheService {

    private StringRedisTemplate stringRedisTemplate;

    @Override
    public String type() {
        return "redis";
    }

    @Override
    public void set(String key, String value, long expiresInSeconds) {
        stringRedisTemplate.opsForValue().set(key, value, expiresInSeconds, TimeUnit.SECONDS);
    }

    @Override
    public boolean exists(String key) {
        return stringRedisTemplate.hasKey(key);
    }

    @Override
    public void delete(String key) {
        stringRedisTemplate.delete(key);
    }

    @Override
    public String get(String key) {
        return stringRedisTemplate.opsForValue().get(key);
    }

    @Override
    public Long increment(String key, long val) {
        return stringRedisTemplate.opsForValue().increment(key,val);
    }
}
```

**配置类**

```java
@Configuration(proxyBeanMethods = false)
@ImportAutoConfiguration(AjCaptchaAutoConfiguration.class)
public class YudaoCaptchaConfiguration {

    @Bean(name = "AjCaptchaCacheService")
    @Primary
    public CaptchaCacheService captchaCacheService(AjCaptchaProperties config,
                                                   StringRedisTemplate stringRedisTemplate) {
        CaptchaCacheService captchaCacheService = CaptchaServiceFactory.getCache(config.getCacheType().name());
        if (captchaCacheService instanceof RedisCaptchaServiceImpl) {
            ((RedisCaptchaServiceImpl) captchaCacheService).setStringRedisTemplate(stringRedisTemplate);
        }
        return captchaCacheService;
    }
}
```
