# 90 - 频率限制（验证码）

> 基于八步法分析 yudao-cloud 中的验证码频率限制实现

---

## ① Why - 价值（为什么）

### 背景与痛点

短信验证码是常见的安全验证手段，但面临以下风险：

| 风险 | 说明 |
|------|------|
| **短信轰炸** | 恶意调用发送验证码，浪费公司短信费用 |
| **暴力破解** | 尝试大量验证码组合破解用户账号 |
| **资源耗尽** | 大量请求导致服务器资源耗尽 |

### 频率限制价值

| 价值 | 说明 |
|------|------|
| **防轰炸** | 限制单用户发送频率，防止恶意发送 |
| **成本控制** | 限制每日发送总量，控制短信费用 |
| **系统保护** | 防止大量请求压垮服务 |

### 适用场景

- 用户注册/登录验证码
- 支付确认验证码
- 敏感操作验证码
- 密码找回验证码

---

## ② What - 定义（是什么）

### 一句话定义

**频率限制**是一种保护机制，通过限制单位时间内的请求次数，防止恶意调用和资源耗尽。

### 核心概念

| 概念 | 说明 |
|------|------|
| **发送频率** | 发送给同一用户的间隔时间 |
| **每日上限** | 每天可发送的最大数量 |
| **IP 限制** | 同一 IP 每天发送上限 |
| **验证码有效期** | 验证码有效时长 |

### yudao-cloud 中的实现

yudao-cloud 使用 **两层限流**：

```
┌─────────────────────────────────────────────────┐
│              验证码频率限制架构                 │
├─────────────────────────────────────────────────┤
│  第1层：SmsCodeService 层                    │
│  - 发送频率限制 (sendFrequency)              │
│  - 每日上限限制 (sendMaximumQuantityPerDay) │
│                                           │
│  第2层：RateLimiter 框架层                 │
│  - @RateLimiter 注解                       │
│  - 基于 Redis 实现                        │
└─────────────────────────────────────────────────┘
```

### 代码实现

#### 发送验证码

```java
public String createSmsCode(String mobile, Integer scene, String ip) {
    // 1. 检查发送频率
    SmsCodeDO lastSmsCode = smsCodeMapper.selectLastByMobile(mobile);
    if (lastSmsCode != null) {
        long interval = Duration.between(lastSmsCode.getCreateTime(), LocalDateTime.now()).toMillis();
        if (interval < smsCodeProperties.getSendFrequency().toMillis()) {
            throw exception(SMS_CODE_SEND_TOO_FAST); // 发送过于频繁
        }
    }
    
    // 2. 检查每日上限
    if (isToday(lastSmsCode.getCreateTime()) && 
        lastSmsCode.getTodayIndex() >= smsCodeProperties.getSendMaximumQuantityPerDay()) {
        throw exception(SMS_CODE_EXCEED_SEND_MAXIMUM_QUANTITY_PER_DAY); // 超过每日上限
    }
    
    // 3. 生成验证码
    String code = String.format("%06d", randomInt(100000, 999999));
    
    // 4. 插入记录
    SmsCodeDO newSmsCode = SmsCodeDO.builder()
        .mobile(mobile)
        .code(code)
        .scene(scene)
        .todayIndex(lastSmsCode != null ? lastSmsCode.getTodayIndex() + 1 : 1)
        .build();
    smsCodeMapper.insert(newSmsCode);
    
    return code;
}
```

#### 验证码校验

```java
public void validateSmsCode(SmsCodeValidateReqDTO reqDTO) {
    // 1. 查询验证码
    SmsCodeDO smsCode = smsCodeMapper.selectLastByMobile(
        reqDTO.getMobile(), reqDTO.getCode(), reqDTO.getScene());
    
    // 2. 验证码是否存在
    if (smsCode == null) {
        throw exception(SMS_CODE_NOT_FOUND);
    }
    
    // 3. 是否已使用
    if (smsCode.getUsed()) {
        throw exception(SMS_CODE_ALREADY_USED);
    }
    
    // 4. 是否过期
    if (Duration.between(smsCode.getCreateTime(), LocalDateTime.now())
        .toMillis() > smsCodeProperties.getExpireTimes().toMillis()) {
        throw exception(SMS_CODE_EXPIRED);
    }
}
```

## ④ Hard - 难点（挑战）

### 难点1：分布式环境下的频率限制

**���题**：多实例部署时，如何保证频率限制准确性？

**场景**：
- 用户请求发送到 Gateway-1
- 验证码保存到 DB-1
- 用户请求发送到 Gateway-2
- 需要读到同一个验证码

**解决方案**：使用数据库 + Redis 结合

```java
// 验证码存储到数据库（保证一致性）
@Resource
private SmsCodeMapper smsCodeMapper;

// 限流使用 Redis（高性能）
@Resource
private RateLimiterRedisDAO rateLimiterRedisDAO;
```

### 难点2：IP 限制

**问题**：同一 IP 下多个手机号，如何限制？

**现状**：yudao-cloud **未实现** IP 级别限制

```java
// TODO: 需要添加 IP 限制
// TODO: 每个 IP 每天可发送数量
// TODO: 每个 IP 每小时可发送数量
```

**建议实现**：

```java
// 使用 Redis 存储 IP 限流
String ipKey = "sms:ip:" + ip + ":" + LocalDate.now();
Long count = redis.opsForValue().increment(ipKey);
if (count != null && count > MAX_IP_PER_DAY) {
    throw exception(SMS_CODE_IP_EXCEED_LIMIT);
}
```

### 难点3：验证码暴力破解

**问题**：攻击者尝试所有验证码组合

**解决方案**：

```
1. 限制校验次数：连续错误5次，锁定10分钟
2. 限制尝试频率：每10秒最多尝试10次
3. 图形验证码：错误3次后需图形验证码
```

## ⑥ Select - 选型（选哪个）

### 候选方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 内存限流 | 性能高 | 多实例不一致 | 单机 |
| 数据库限流 | 一致性好 | 性能低 | 中小流量 |
| Redis限流 | 性能高+一致 | 需要Redis | 大流量 |
| 分布式限流 | 精确控制 | 实现复杂 | 超大流量 |

### yudao-cloud 选型

| 组件 | 方案 | 说明 |
|------|------|------|
| 验证码频率 | 数据库 | 使用 SmsCodeDO 表 |
| 接口限流 | Redis | @RateLimiter 框架 |
| IP限制 | 未实现 | 需手动添加 |

## 附录：限流算法

### 令牌桶算法

```
┌────────────────��────────────────────┐
│           令牌桶算法                │
├─────────────────────────────────────┤
│  1. 固定速率往桶中添加令牌          │
│  2. 获取令牌时减去一个令牌         │
│  3. 桶满时丢弃多余令牌            │
│  4. 无令牌时拒绝请求              │
└─────────────────────────────────────┘
```

### 滑动窗口算法

```
┌─────────────────────────────────────┐
│          滑动窗口算法               │
├─────────────────────────────────────┤
│  1. 记录每个请求的时间戳            │
│  2. 统计窗口内的请求数              │
│  3. 超限时拒绝请求                 │
│  4. 窗口滑动时清理过期记录         │
└─────────────────────────────────────┘
```
