# 05-通知模板

> 通知模板模块是消息通知系统中，实现模板化管理与参数替换的核心功能。本文档按照八步法进行分析。

---

## ① Why - 价值 (为什么)

### 背景与痛点
- **业务场景**：运营需要灵活配置通知内容，如订单通知、系统公告、营销消息等
- **痛点**：硬编码通知内容，修改需改代码、重新发布
- **需求**：模板化管理，支持参数动态替换

### 价值收益
- ✅ **灵活性**：运营可后台配置模板，修改即时生效
- ✅ **复用性**：同一模板可发给不同用户，参数不同
- ✅ **可维护性**：模板统一管理，无需代码修改

### 使用者
- 运营人员（配置模板）
- 开发人员（调用发送接口）
- 最终用户（接收通知）

---

## ② What - 定义 (是什么)

### 核心概念
**通知模板**：支持参数占位符的通知内容配置，支持动态内容生成。

### 核心组成部分
1. **NotifyTemplateDO** - 模板实体
2. **NotifyTemplateMapper** - 数据访问
3. **NotifyTemplateService** - 模板管理CRUD
4. **parseTemplateContentParams()** - 参数解析
5. **formatNotifyTemplateContent()** - 内容格式化

### 关键术语
| 术语 | 说明 |
|------|------|
| 占位符 | 模板中 `{name}` 格式的参数 |
| 参数解析 | 从模板内容提取所有占位符 |
| 参数替换 | 将占位符替换为实际值 |
| 模板编码 | 模板唯一标识 |

### 关键代码设计

#### 1. 参数解析 (正则匹配)
```java
private static final Pattern PATTERN_PARAMS = Pattern.compile("\\{(.*?)}");

public List<String> parseTemplateContentParams(String content) {
    return ReUtil.findAllGroup1(PATTERN_PARAMS, content);
}

// 示例：content = "您好 {name}，订单 {orderNo} 已发货"
// 结果：["name", "orderNo"]
```

#### 2. 内容格式化
```java
public String formatNotifyTemplateContent(String content, Map<String, Object> params) {
    return StrUtil.format(content, params);
}

// 示例：content = "您好 {name}"，params = {name: "张三"}
// 结果："您好 张三"
```

#### 3. 缓存策略
```java
@Cacheable(cacheNames = RedisKeyConstants.NOTIFY_TEMPLATE, key = "#code", unless = "#result == null")
public NotifyTemplateDO getNotifyTemplateByCodeFromCache(String code) {
    return notifyTemplateMapper.selectByCode(code);
}

// 修改/删除时清空所有缓存
@CacheEvict(cacheNames = RedisKeyConstants.NOTIFY_TEMPLATE, allEntries = true)
```

## ④ Hard - 难点 (挑战)

### 难点1：参数自动发现
```java
// 场景：运营输入模板内容 "您好 {name}"
// 难点：系统需要自动识别参数列表
public List<String> parseTemplateContentParams(String content) {
    // 使用正则匹配 {xxx} 格式
    return ReUtil.findAllGroup1(PATTERN_PARAMS, content);
}
// 边界：{name}、{order_no}、{} (空参数)
```

### 难点2：模板编码唯一性
```java
// 场景：创建模板时编码重复
// 难点：需要校验唯一性，同时考虑更新场景
public void validateNotifyTemplateCodeDuplicate(Long id, String code) {
    NotifyTemplateDO template = notifyTemplateMapper.selectByCode(code);
    if (template == null) return;
    // id为空 → 新增场景，直接报错
    // id不为空 → 更新场景，非自身则报错
    if (id == null || !template.getId().equals(id)) {
        throw exception(NOTIFY_TEMPLATE_CODE_DUPLICATE, code);
    }
}
```

### 难点3：缓存一致性
```java
// 场景：修改模板后缓存需要更新
// 难点：缓存key是code，但修改用id
@CacheEvict(cacheNames = RedisKeyConstants.NOTIFY_TEMPLATE, allEntries = true)
// 使用 allEntries = true 清空所有缓存，简单粗暴但有效
```

### 难点4：参数校验时机
```java
// 场景：发送时参数缺失
// 难点：需要在校验模板时就检查参数
public void validateTemplateParams(NotifyTemplateDO template, Map<String, Object> templateParams) {
    template.getParams().forEach(key -> {
        Object value = templateParams.get(key);
        if (value == null) {
            throw exception(NOTIFY_SEND_TEMPLATE_PARAM_MISS, key);
        }
    });
}
```

## ⑥ Select - 选型 (选哪个)

### 候选方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 自研模板系统 | 完全可控、可定制 | 需要开发 | 企业内部系统 |
| 模板引擎(Freemarker) | 功能强大、社区成熟 | 学习成本 | 复杂模板场景 |
| 占位符简单替换 | 实现简单、性能高 | 功能有限 | 简单通知场景 |

### 项目选型
**yudao-cloud 选择自研简单模板**，因为：
1. 功能简单，占位符 `{key}` 足够
2. 性能高，使用 Hutool 的 StrUtil.format
3. 与项目深度集成，缓存策略可控
4. 维护简单，无额外依赖

### 核心流程步骤

```
Step 1: 创建模板
  → 校验点：code 唯一
  → 验证方法：selectByCode 返回null

Step 2: 解析参数
  → 校验点：正则匹配 {xxx}
  → 验证方法：parseTemplateContentParams

Step 3: 更新模板
  → 校验点：记录存在
  → 验证方法：selectById 非null

Step 4: 清空缓存
  → 校验点：allEntries = true
  → 验证方法：Redis key 删除

Step 5: 格式化内容
  → 校验点：占位符替换完成
  → 验证方法：StrUtil.format 返回
```

### 失败恢复机制

```
场景1：模板编码重复
  → 触发：selectByCode 返回非null
  → 恢复：抛出 NOTIFY_TEMPLATE_CODE_DUPLICATE

场景2：模板不存在
  → 触发：selectById 返回null
  → 恢复：抛出 NOTIFY_TEMPLATE_NOT_EXISTS

场景3：参数缺失
  → 触发：params.get(key) == null
  → 恢复：抛出 NOTIFY_SEND_TEMPLATE_PARAM_MISS
```

## ⑧ SKILL - 提炼 (复用)

### 触发条件
```
场景1：需要配置通知模板
场景2：需要动态内容生成
场景3：需要参数校验
```

### 执行流程
```
Step 1: 创建模板
  - POST /notify/template/create
  - 传入 name, code, type, nickname, content, status

Step 2: 自动解析参数
  - 后台自动解析 content 中的 {xxx}
  - 存储到 params 字段

Step 3: 调用发送
  - POST /notify/template/send
  - 传入 templateCode, userId, userType, params

Step 4: 参数校验与替换
  - 校验 params 包含所有必填参数
  - StrUtil.format 替换占位符
```
