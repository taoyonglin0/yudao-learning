# 21-第三方登录

> 认证登录 - 第三方登录接入

---

## ① Why - 价值

**背景与痛点**：
- 用户不想注册新账号，流失率高
- 手动注册流程繁琐，用户体验差
- 密码管理不安全，忘记密码问题多

**收益**：
- 一键登录，用户转化率提升 30%+
- 无需存储密码，安全风险降低
- 支持多平台：微信、GitHub、钉钉等

**用户**：
- C 端用户、前端开发者

---

## ② What - 定义

**一句话定义**：
第三方登录是通过 OAuth2.0 协议，让用户使用已有社交账号（微信/GitHub/钉钉等）登录系统，无需注册新账号。

**核心组成**：
1. **社交平台** - 第三方账号提供方（微信/GitHub/钉钉）
2. **社交用户** - 第三方平台的用户信息
3. **绑定关系** - 社交用户与本系统用户的关联
4. **JustAuth** - 第三方登录集成库

**关键术语**：
- OAuth2.0 - 授权协议
- Code - 授权码（临时凭证）
- OpenID - 平台用户唯一标识
- UnionID - 跨应用用户唯一标识（微信）

**目标**：实现第三方账号登录/绑定

**范围**：
- 允许：yudao-module-system/（系统模块）
- 禁止：改动业务模块

**数据模型**：

```java
// 社交用户表 (system_social_user)
SocialUserDO:
  - id              : 编号
  - userId          : 用户编号（可空，未绑定时）
  - socialType      : 社交平台类型(10=Gitee 20=钉钉 30=企业微信 31=公众号 32=微信开放平台 34=微信小程序 40=支付宝)
  - code            : 授权码
  - state           : 状态参数（防CSRF）
  - openid          : 平台用户唯一标识
  - token           : 访问令牌
  - rawTokenInfo    : 原始令牌信息(JSON)
  - nickname        : 昵称
  - avatar          : 头像
  - rawUserInfo     : 原始用户信息(JSON)

 // 用户绑定表 (system_social_user_bind)
SocialUserBindDO:
  - id              : 编号
  - userId          : 用户编号
  - userType        : 用户类型(1=管理员 2=会员)
  - socialUserId    : 社交用户编号
  - socialType      : 社交平台类型
```

**核心流程**：

```
流程1：扫码授权
用户点击第三方登录 → 跳转第三方授权页面 → 用户授权 → 回调带 code → 后端用 code 换 token → 获取用户信息

流程2：绑定已有账号
用户已登录 → 点击第三方绑定 → 扫码授权 → 绑定社交用户到当前账号

流程3：自动注册登录
新用户扫码授权 → 系统创建社交用户 → 自动注册账号 → 登录成功
```

## ④ Hard - 难点

**难点1：同一用户多平台识别？**
```
场景：用户用微信和 GitHub 登录，如何识别是同一人？
→ 目前不支持跨平台绑定，需要用户手动绑定
→ 微信体系内：UnionID 可跨公众号/小程序识别
```

**难点2：Token 过期处理？**
```
场景：第三方平台 Token 过期了怎么办？
→ 引导用户重新授权
→ 系统保存 refresh_token（部分平台支持）
```

**难点3：安全性（CSRF）？**
```
场景：恶意攻击者伪造授权回调？
→ 使用 state 参数验证回调来源
→ 每次授权生成随机 state，验证一致性
```

**难点4：解绑后重新绑定？**
```
场景：用户解绑微信后想重新绑定？
→ 解绑时删除绑定关系，保留社交用户记录
→ 重新授权时可绑定同一社交账号
```

**核心代码**：

```java
// SocialUserServiceImpl - 绑定社交用户
@Override
@Transactional(rollbackFor = Exception.class)
public String bindSocialUser(SocialUserBindReqDTO reqDTO) {
    // 1. 通过 code 获取社交用户信息
    SocialUserDO socialUser = authSocialUser(reqDTO.getSocialType(), 
            reqDTO.getUserType(), reqDTO.getCode(), reqDTO.getState());
    
    // 2. 解绑之前的绑定（如果有）
    socialUserBindMapper.deleteByUserTypeAndSocialUserId(reqDTO.getUserType(), 
            socialUser.getId());
    socialUserBindMapper.deleteByUserTypeAndUserIdAndSocialType(reqDTO.getUserType(), 
            reqDTO.getUserId(), socialUser.getType());
    
    // 3. 绑定当前用户
    SocialUserBindDO socialUserBind = SocialUserBindDO.builder()
            .userId(reqDTO.getUserId())
            .userType(reqDTO.getUserType())
            .socialUserId(socialUser.getId())
            .socialType(socialUser.getType())
            .build();
    socialUserBindMapper.insert(socialUserBind);
    return socialUser.getOpenid();
}
```

```java
// 获取社交用户（核心）
private SocialUserDO authSocialUser(Integer socialType, Integer userType, 
        String code, String state) {
    // 1. 构建授权Url
    AuthSource source = SocialTypeEnum.valueOfType(socialType).getSource();
    AuthRequest authRequest = AuthRequestBuilder.getInstance().build(source, config);
    
    // 2. 获取授权结果（内部自动用 code 换 token 和用户信息）
    AuthUser authUser = authRequest.login(AuthCallback.builder()
            .code(code).state(state).build());
    
    // 3. 保存或更新社交用户
    return saveOrUpdateSocialUser(authUser);
}
```
