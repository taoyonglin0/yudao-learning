# 技术学习框架 - 八步法 (实战版)

> 每个课题按此框架执行，确保学到手、能落地

---

## ① Why - 价值 (为什么)

**目标**：解决什么问题？带来什么价值？

**必答**：
- 背景是什么？不用它有什么痛点？
- 用了它能带来什么收益？（效率↑/成本↓/体验↑）
- 谁会用到这个功能？

**大白话案例**：
```
比如：原来每天手动同步数据到凌晨 3 点，用了定时任务后，
系统自动跑，第二天上班数据已经同步好了，运营可以直接看报表。
```

---

## ② What - 定义 (是什么)

**目标**：核心概念是什么？

**必答**：
- 一句话定义这个技术/功能
- 核心组成部分有哪些？
- 关键术语有哪些？

**大白话案例**：
```
XXL-Job就是一个分布式定时任务调度平台，
有3个核心：调度中心(管理任务)、执行器(跑任务)、任务(具体业务)
```

---

## ③ How - 思维 (怎么做)

**目标**：如果我们自己做，怎么实施？数据模型？

**必答**：
- **数据模型**：要设计几张表？每张表有什么字段？
- 核心流程是什么？(从入口到结果)
- 关键方法/类有哪些？

```
目标：我想要实现/修复什么？
范围：只允许动哪些目录/文件？
  例：yudao-module-job/, yudao-framework/yudao-spring-boot-starter-job/
禁止：哪些目录/文件/行为绝对不要碰？
  例：禁止修改 yudao-module-system/ 下的任何文件
```

**大白话案例**：
```
我要设计一个定时任务表，最少需要：
- job_id (任务ID)
- cron_expression (cron表达式)
- handler名称 (哪个类的方法)
- status (启用/停用)

流程：调度中心读取cron → 到了时间 → 调用执行器 → 执行Job → 记录日志
```

## ④ Hard - 难点 (挑战)

**目标**：挑战和坑在哪？举例子

**必答**：
- 可能遇到什么问题？
- 边界情况有哪些？
- 有什么坑？

**大白话案例**：
```
难点1：多台机器同时跑，任务重复执行？
  → 需要分布式锁，只有一台能抢到

难点2：任务执行一半挂了？
  → 需要补偿机制，自动重试

难点3：想改任务参数不想重启？
  → 需要热更新，从数据库读配置
```

## ⑤ Metric - 衡量 (指标)

**目标**：衡量这个功能做得好不好的指标

**指标设计原则**：
- 不是固定指标，是基于**这个业务功能**设计的指标
- 每个指标都要有明确的**权重**

**指标维度**：
| 指标 | 权重 | 说明 | 怎么验证 |
|------|------|------|----------|
| 功能完整性 | 20% | 需求覆盖程度 | 逐条核对需求 |
| 可用性 | 20% | 系统稳定运行 | 24h运行无报错 |
| 性能 | 15% | 执行时间/吞吐 | 压测报告 |
| 可维护性 | 15% | 代码清晰度 | 可读性review |
| 扩展性 | 15% | 能否灵活扩展 | 新需求开发时间 |
| 安全性 | 15% | 权限/数据安全 | 安全测试 |

**大白话案例**：
```
定时任务衡量指标：
- 任务执行成功率 ≥ 99.9% (权重30%)
- 任务重复执行次数 = 0 (权重25%)
- 任务修改生效时间 ≤ 1min (权重20%)
- 任务执行日志可查 (权重15%)
- 支持动态修改不重启 (权重10%)
```

## ⑦ Impl - 实现 (细节)

**目标**：项目内具体实现是什么？

**必答**：
- **数据模型**：实体类/SQL
- **核心代码**：关键方法
- **算法**：核心逻辑
- **组件**：用到什么技术组件

```
关键步骤校验：
  Step 1: xxx → 校验点：xxx
  Step 2: xxx → 校验点：xxx
  校验机制：单元测试 / 集成测试

失败恢复：
  - 如果xxx失败 → 触发xxx机制 → 回滚到xxx状态

约束：
  - 不引入新依赖？
  - 保持 API 兼容？
  - 性能要求：P99 < xxxms？
  - 安全要求：需要脱敏？
```

**大白话案例**：
```
数据模型：
class XxlJob {
    private Long id;
    private String name;
    private String cron;
    private String handler;
    private Integer status;
}

核心代码：
@XxlJob("demoJob")
public void execute() {
    // 1. 执行业务逻辑
    // 2. 记录日志
}
```

**证据**：分析都要有明确的代码/文档参考，列出引用的文件路径
