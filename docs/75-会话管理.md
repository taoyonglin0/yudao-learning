# 75-会话管理 - 大模型接入之会话管理

## ① Why - 价值 (为什么)

### 背景与痛点

在大模型聊天应用中，企业面临以下挑战：

1. **上下文丢失**：每次请求独立，无法记忆对话历史
2. **体验碎片化**：用户需要重复说明背景信息
3. **多会话混乱**：用户可能有多个聊天主题，缺乏组织
4. **上下文膨胀**：长对话导致 Token 超出限制

### 带来的价值

- ✅ **上下文记忆**：自动维护对话历史，跨越多轮交互
- ✅ **会话隔离**：支持多主题并行对话，互不干扰
- ✅ **智能摘要**：自动管理上下文长度，避免超出限制
- ✅ **状态持久化**：会话数据持久化存储，跨设备同步

### 使用者

- C端用户（与 AI 助手进行多轮对话）
- 后端开发工程师
- AI 应用开发者

## ② What - 定义 (是什么)

### 一句话定义

**会话管理**是指对 AI 聊天会话的创建、消息存储、上下文维护、状态管理进行系统化管理的技术方案。

### 核心组成部分

| 组件 | 说明 |
|------|------|
| **Conversation（会话）** | 一次完整的聊天会话，包含标题、配置、消息列表 |
| **Message（消息）** | 会话中的单条消息，包含角色、内容、时间戳 |
| **Context（上下文）** | 传递给模型的对话历史 |
| **配置（ temperature、maxTokens）** | 会话级别的大模型参数 |

### 关键术语

- **System Message**：系统设定 Prompt
- **User Message**：用户消息
- **Assistant Message**：AI 回复消息
- **Temperature**：温度参数，控制随机性
- **Max Tokens**：单次回复最大 Token 数
- **Max Contexts**：上下文最大消息数

## ④ Hard - 难点 (挑战)

### 难点 1：上下文长度限制

模型有上下文窗口限制（如 128K tokens），长对话会超出限制。

**解决方案**：
- 保留最近 N 轮消息
- 消息摘要压缩
- 滑动窗口策略

### 难点 2：Token 成本控制

全量历史消息浪费 Token，增加成本。

**解决方案**：
- 按 maxContexts 限制
- 只传递相关上下文
- 提示词精简

### 难点 3：多会话数据隔离

确保用户只能访问自己的会话。

**解决方案**：
- userId 关联检查
- 租户隔离
- 权限校验

### 难点 4：消息检索效率

消息量大时查询慢。

**解决方案**：
- 索引优化
- 分页查询
- 缓存策略

## ⑥ Select - 选型 (选哪个)

### 候选方案

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **内存缓存** | 速度快 | 断电丢失、重启丢失 | 临时测试 |
| **数据库存储** | 持久化、支持复杂查询 | 有延迟 | 生产环境 |
| **Redis缓存+DB** | 兼顾速度和持久化 | 架构复杂 | 高并发场景 |

### 选型理由

选择 **数据库存储**，因为：

1. ✅ **持久化**：确保会话数据不丢失
2. ✅ **查询灵活**：支持分页、搜索等多种查询
3. ✅ **事务支持**：保证数据一致性
4. ✅ **与项目技术栈一致**：使用 MyBatis Plus

### 核心代码 - 创建会话

```java
@Service
public class AiChatConversationServiceImpl {
    
    @Resource
    private AiChatConversationMapper chatConversationMapper;
    @Resource
    private AiChatRoleService chatRoleService;
    @Resource
    private AiModelService modalService;
    
    @Override
    public Long createChatConversationMy(AiChatConversationCreateMyReqVO createReqVO, Long userId) {
        // 1. 获取角色和模型
        AiChatRoleDO role = createReqVO.getRoleId() != null 
            ? chatRoleService.validateChatRole(createReqVO.getRoleId()) : null;
        
        AiModelDO model = role != null && role.getModelId() != null 
            ? modalService.validateModel(role.getModelId())
            : modalService.getRequiredDefaultModel(AiModelTypeEnum.CHAT.getType());
        
        // 2. 创建会话
        AiChatConversationDO conversation = new AiChatConversationDO()
            .setUserId(userId)
            .setModelId(model.getId())
            .setModel(model.getModel())
            .setTemperature(model.getTemperature())
            .setMaxTokens(model.getMaxTokens())
            .setMaxContexts(model.getMaxContexts());
            
        if (role != null) {
            conversation.setTitle(role.getName())
                .setRoleId(role.getId())
                .setSystemMessage(role.getSystemMessage());
        } else {
            conversation.setTitle(AiChatConversationDO.TITLE_DEFAULT);
        }
        
        chatConversationMapper.insert(conversation);
        return conversation.getId();
    }
}
```

### 核心代码 - 消息服务

```java
@Service
public class AiChatMessageServiceImpl {
    
    public String chat(AiChatMessageDO message) {
        // 1. 获取会话
        AiChatConversationDO conversation = conversationService.getChatConversation(conversationId);
        
        // 2. 构建消息列表
        List<ChatMessage> chatMessages = new ArrayList<>();
        
        // 2.1 添加 System Prompt
        if (StrUtil.isNotBlank(conversation.getSystemMessage())) {
            chatMessages.add(new SystemMessage(conversation.getSystemMessage()));
        }
        
        // 2.2 添加历史消息
        List<AiChatMessageDO> historyMessages = getHistoryMessages(conversationId, conversation.getMaxContexts());
        for (AiChatMessageDO historyMsg : historyMessages) {
            if (MessageType.USER.getValue().equals(historyMsg.getType())) {
                chatMessages.add(new UserMessage(historyMsg.getContent()));
            } else if (MessageType.ASSISTANT.getValue().equals(historyMsg.getType())) {
                chatMessages.add(new AssistantMessage(historyMsg.getContent()));
            }
        }
        
        // 2.3 添加当前用户消息
        chatMessages.add(new UserMessage(message.getContent()));
        
        // 3. 获取模型
        AiModelDO model = modelService.getModel(conversation.getModelId());
        ChatModel chatModel = aiModelFactory.getOrCreateChatModel(
            AiPlatformEnum.valueOf(model.getPlatform()),
            getApiKey(model.getKeyId()),
            model.getUrl()
        );
        
        // 4. 调用模型
        ChatResponse response = chatModel.call(new Prompt(chatMessages));
        String replyContent = response.getResult().getOutput().getText();
        
        // 5. 保存消息和回复
        message.setType(MessageType.USER.getValue());
        message.setConversationId(conversationId);
        chatMessageMapper.insert(message);
        
        AiChatMessageDO replyMessage = new AiChatMessageDO();
        replyMessage.setConversationId(conversationId);
        replyMessage.setReplyId(message.getId());
        replyMessage.setType(MessageType.ASSISTANT.getValue());
        replyMessage.setContent(replyContent);
        replyMessage.setModel(model.getModel());
        replyMessage.setModelId(model.getId());
        chatMessageMapper.insert(replyMessage);
        
        return replyContent;
    }
    
    private List<AiChatMessageDO> getHistoryMessages(Long conversationId, Integer maxContexts) {
        return chatMessageMapper.selectList(
            new LambdaQueryWrapper<AiChatMessageDO>()
                .eq(AiChatMessageDO::getConversationId, conversationId)
                .orderByDesc(AiChatMessageDO::getId)
                .last("LIMIT " + maxContexts)
        );
    }
}
```

## ⑧ SKILL - 提炼 (复用)

### 触发条件

- 场景 1：需要在 AI 聊天中维护多轮对话上下文
- 场景 2：需要管理多个独立的聊天会话
- 场景 3：需要支持会话标题、置顶等功能
- 场景 4：需要跨设备同步对话历史

### 执行流程

```
Step 1: 创建会话
  - 用户首次聊天或点击"新建会话"
  - 系统创建 AiChatConversationDO 记录
  - 返回会话ID

Step 2: 发送消息
  - 用户在会话中发送消息
  - 系统加载该会话的历史消息
  - 组装上下文调用 ChatModel

Step 3: 保存消息
  - AI 回复后保存用户消息和AI回复
  - 消息关联会话ID

Step 4: 管理会话
  - 支持修改标题、置顶��删除
  - 支持分页查看历史消息
```
