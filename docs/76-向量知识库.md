# 76-向量知识库 - AI应用之向量知识库

> 本文档按照八步法（Why-What-How-Hard-Metric-Select-Impl-SKILL）分析向量知识库技术

## ① Why - 价值 (为什么)

### 背景与痛点

在大模型应用场景中，企业面临以下挑战：

1. **知识更新滞后**：模型训练数据有时效性，无法获取最新信息
2. **幻觉问题**：LLM 可能产生错误或不准确的回答
3. **专业领域知识不足**：通用模型缺乏特定行业的专业知识
4. **数据安全顾虑**：不希望将内部文档上传到第三方

### 带来的价值

- ✅ **知识时效性**：可以随时更新知识库，保持最新信息
- ✅ **减少幻觉**：基于实际文档回答，可追溯来源
- ✅ **专业领域适配**：构建行业专属知识库，提升专业度
- ✅ **数据自主可控**：本地部署，知识不外泄

### 使用者

- 企业内部知识管理
- 客服系统（产品手册、FAQ）
- 专业领域助手（法律、医疗、金融）
- RAG 应用开发者

## ② What - 定义 (是什么)

### 一句话定义

**向量知识库**是将文档切分为语义片段，通过 Embedding 模型转换为向量，存储到向量数据库中，实现语义相似度检索的技术方案。

### 核心组成部分

| 组件 | 说明 |
|------|------|
| **Knowledge（知识库）** | 文档集合的容器，管理知识库配置 |
| **Document（文档）** | 上传的原始文档（PDF、Word、Markdown 等） |
| **Segment（分段/切片）** | 文档经过切分后的语义单元 |
| **Embedding（向量化）** | 将文本转换为向量表示 |
| **Vector Store（向量库）** | 存储向量数据的数据库（Milvus、Qdrant 等） |

### 关键术语

- **Embedding**：将文本映射为稠密向量
- **Vector Store**：向量数据库，存储和检索向量
- **TopK**：返回最相似的 K 条结果
- **Similarity Threshold**：相似度阈值，过滤低相关度结果
- **Rerank**：对检索结果重新排序，提升准确率
- **Text Splitter**：文本分割器，将长文本切分为小块

## ④ Hard - 难点 (挑战)

### 难点 1：文档解析

不同格式（PDF、Word、HTML）解析困难，提取文本不完整。

**解决方案**：
- 使用专业解析库（Apache Tika、PDFBox）
- 针对不同格式采用不同策略
- 保留关键格式信息（标题、列表）

### 难点 2：文本分割

如何合理切分文档，既保持语义完整，又不会信息丢失。

**解决方案**：
- 固定长度分割（如 500 字符）
- 语义分割（按段落、主题）
- 重叠分割（保留上下文）

### 难点 3：向量检索效果

简单向量检索可能遗漏相关内容或召回噪音。

**解决方案**：
- 调整 TopK 和相似度阈值
- 使用 Rerank 模型重排
- 多路召回策略

### 难点 4：向量库选型

Milvus、Qdrant、Redis、Elasticsearch 各有优劣。

**解决方案**：
- 根据规模选择（中小规模可用 Redis）
- 根据功能选择（需要复杂查询选 Elasticsearch）
- 根据部署要求选择（国产化选 Milvus）

## ⑥ Select - 选型 (选哪个)

### 候选方案

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **Milvus** | 功能强大、社区活跃、国产 | 资源占用大 | 大规模生产环境 |
| **Qdrant** | 性能高、易用性好 | 社区相对较小 | 中小规模 |
| **Redis** | 速度快、部署简单 | 功能有限 | 简单场景、快速上线 |
| **Elasticsearch** | 生态丰富、支持复杂查询 | 资源占用大 | 需要全文检索 |

### 选型理由

选择 **Spring AI VectorStore 抽象 + Milvus/Qdrant**，因为：

1. ✅ **标准化接口**：Spring AI 提供统一 VectorStore 接口
2. ✅ **多向量库支持**：可灵活切换底层存储
3. ✅ **国产化支持**：Milvus 国产开源，可信可控
4. ✅ **功能完整**：支持相似度检索、过滤、聚合

### 工具集

- **向量库**：Milvus、Qdrant、Redis、Elasticsearch
- **Embedding 模型**：OpenAI、Azure OpenAI、通义千问、智谱
- **文档解析**：Apache Tika、PDFBox
- **官方文档**：https://docs.spring.io/spring-ai/reference/

```java
@Service
public class AiKnowledgeSegmentServiceImpl {
    
    @Resource
    private AiModelService modelService;
    
    private VectorStore getVectorStoreById(AiKnowledgeDO knowledge) {
        // 获取向量模型
        AiModelDO embeddingModel = modelService.getModel(knowledge.getEmbeddingModelId());
        EmbeddingModel model = aiModelFactory.getOrCreateEmbeddingModel(
            AiPlatformEnum.valueOf(embeddingModel.getPlatform()),
            getApiKey(embeddingModel.getKeyId()),
            embeddingModel.getUrl(),
            embeddingModel.getModel()
        );
        
        // 构建元数据字段类型
        Map<String, Class<?>> metadataFields = Map.of(
            VECTOR_STORE_METADATA_KNOWLEDGE_ID, String.class,
            VECTOR_STORE_METADATA_DOCUMENT_ID, String.class,
            VECTOR_STORE_METADATA_SEGMENT_ID, String.class
        );
        
        // 创建向量库（Milvus/Qdrant/Redis）
        return aiModelFactory.getOrCreateVectorStore(
            MilvusVectorStore.class,
            model,
            metadataFields
        );
    }
}
```

### 核心代码 - 文档分割与向量化

```java
@Override
public void createKnowledgeSegmentBySplitContent(Long documentId, String content) {
    // 1. 获取知识库配置
    AiKnowledgeDocumentDO document = knowledgeDocumentService.validateKnowledgeDocumentExists(documentId);
    AiKnowledgeDO knowledge = knowledgeService.validateKnowledgeExists(document.getKnowledgeId());
    
    // 2. 获取向量库
    VectorStore vectorStore = getVectorStoreById(knowledge);
    
    // 3. 文本分割
    TextSplitter textSplitter = new TokenTextSplitter();
    List<String> chunks = textSplitter.split(content);
    
    // 4. 向量化并存储
    for (String chunk : chunks) {
        Document document = Document.builder()
            .id(UUID.randomUUID().toString())
            .text(chunk)
            .metadata(Map.of(
                "knowledgeId", String.valueOf(knowledge.getId()),
                "documentId", String.valueOf(documentId)
            ))
            .build();
        
        vectorStore.add(List.of(document));
        
        // 5. 保存分段记录到数据库
        AiKnowledgeSegmentDO segment = new AiKnowledgeSegmentDO()
            .setKnowledgeId(knowledge.getId())
            .setDocumentId(documentId)
            .setContent(chunk)
            .setContentLength(chunk.length())
            .setVectorId(document.getId())
            .setStatus(CommonStatusEnum.ENABLE.getStatus());
        segmentMapper.insert(segment);
    }
}
```

### 核心代码 - 相似度检索

```java
@Override
public List<AiKnowledgeSegmentSearchRespBO> searchSegment(AiKnowledgeSegmentSearchReqBO reqBO) {
    // 1. 获取知识库配置
    AiKnowledgeDO knowledge = knowledgeService.validateKnowledgeExists(reqBO.getKnowledgeId());
    
    // 2. 获取向量库
    VectorStore vectorStore = getVectorStoreById(knowledge);
    
    // 3. 构建检索请求
    SearchRequest searchRequest = SearchRequest.builder()
        .query(reqBO.getQuery())
        .topK(reqBO.getTopK() != null ? reqBO.getTopK() : knowledge.getTopK())
        .similarityThreshold(SIMILARITY_THRESHOLD_ACCEPT_ALL)
        .filterExpression(new FilterExpressionBuilder()
            .eq("knowledgeId", String.valueOf(knowledge.getId()))
            .build())
        .build();
    
    // 4. 执行检索
    List<Document> documents = vectorStore.similaritySearch(searchRequest);
    
    // 5. Rerank 重排（如配置）
    if (rerankModel != null) {
        documents = rerank(reqBO.getQuery(), documents, knowledge.getTopK());
    }
    
    // 6. 转换结果
    return documents.stream()
        .map(doc -> {
            AiKnowledgeSegmentSearchRespBO result = new AiKnowledgeSegmentSearchRespBO();
            result.setContent(doc.getText());
            result.setDocumentId(Long.parseLong(doc.getMetadata().get("documentId")));
            result.setSegmentId(Long.parseLong(doc.getMetadata().get("segmentId")));
            result.setScore(1.0); // 简化处理
            return result;
        })
        .collect(Collectors.toList());
}
```
