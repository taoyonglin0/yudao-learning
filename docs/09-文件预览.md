# 09-文件预览.md

> 文件存储 - 文件预览与下载

---

## ① Why - 价值 (为什么)

**背景与痛点**

- **背景**：用户上传文件后，需要能够在线预览（查看图片、PDF等）或下载到本地
- **不用它的痛点**：
  - 只能下载后本地查看，不方便
  - 大文件下载浪费带宽和时间
  - 无法快速确认文件内容是否正确
- **收益**：
  - 在线预览提升用户体验
  - 减少不必要下载，节省带宽
  - 快速确认文件内容

**目标用户**

- 管理后台用户（上传/查看附件）
- 移动端/前端用户（查看预览图）

---

## ② What - 定义 (是什么)

**一句话定义**

文件预览与下载是指用户上传文件后，系统提供在线查看（图片直接显示、PDF/Office在线预览）或完整下载到本地的能力。

**核心组成**

1. **存储模式**
   - 本地存储（Local）
   - 对象存储（OSS/S3/七牛）
   - 数据库存储（DB）
   - FTP/SFTP

2. **访问方式**
   - 直连下载（/infra/file/{configId}/get/{path}）
   - 预签名地址（Presigned URL）
   - 流式预览（attachment/inline）

3. **关键术语**
   - `FileStorageEnum`：存储类型枚举
   - `presignPutUrl`：预签名上传地址
   - `presignGetUrl`：预签名读取地址
   - `Content-Disposition`：下载方式（attachment=inline）

## ③ How - 思维 (怎么做)

### 数据模型设计

```
文件配置表（infra_file_config）：
- id           : 主键
- name         : 配置名称
- storage      : 存储类型（1-DB,10-Local,11-FTP,12-SFTP,20-S3）
- config       : 存储配置（JSON）
- primary      : 是否主存储
- created_time : 创建时间

文件表（infra_file）：
- id           : 主键
- config_id    : 配置ID
- name         : 文件名
- path         : 文件路径
- content_type : MIME类型
- size         : 文件大小
- created_time : 创建时间
```

### 关键流程设计

**下载/预览流程**：
```
用户请求 → /infra/file/{configId}/get/{path} → 
获取文件内容 → 判断类型 → 
writeAttachment（inline=预览，attachment=下载）
```

## ④ Hard - 难点 (挑战)

### 问题1：大文件处理

```
场景：文件几百MB，内存加载OOM
→ 解决：流式读取、分片传输
```

### 问题2：中文文件名

```
场景：文件名含中文，URL编码问题
→ 解决：URLUtil.decode(path, UTF-8)
```

### 问题3：重复文件

```
场景：同名文件覆盖还是保留？
→ 解决：UUID生成唯一路径，或配置覆盖策略
```

### 问题4：权限控制

```
场景：敏感文件需要登录才能下载
→ 解决：去掉@PermitAll，添加权限校验
```

## ⑥ Select - 选型 (选哪个)

### 候选方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 直连存储 | 简单快速 | 权限难控 | 公开文件 |
| 预签名URL | 安全灵活 | 有时效 | 私有文件 |
| 流式代理 | 权限可控 | 占用带宽 | 高安全场景 |
| 第三方预览 | 功能强大 | 依赖外部 | Office预览 |

### 选型结论

**选预签名URL + 本地直连**，因为：
1. 简单高效，CDN加速
2. 结合权限校验控制访问
3. 支持多种存储后端

## ⑦ Impl - 实现 (细节)

### 核心代码

**文件Controller**

```java
@GetMapping("/{configId}/get/**")
@PermitAll
@TenantIgnore
@Operation(summary = "下载文件")
public void getFileContent(HttpServletRequest request,
                           HttpServletResponse response,
                           @PathVariable("configId") Long configId) throws Exception {
    // 获取请求的路径
    String path = StrUtil.subAfter(request.getRequestURI(), "/get/", false);
    if (StrUtil.isEmpty(path)) {
        throw new IllegalArgumentException("结尾的 path 路径必须传递");
    }
    // 解码，解决中文路径的问题
    path = URLUtil.decode(path, StandardCharsets.UTF_8, false);

    // 读取内容
    byte[] content = fileService.getFileContent(configId, path);
    if (content == null) {
        log.warn("[getFileContent][configId({}) path({}) 文件不存在]", configId, path);
        response.setStatus(HttpStatus.NOT_FOUND.value());
        return;
    }
    writeAttachment(response, path, content);
}
```

**FileStorageEnum**

```java
public enum FileStorageEnum {
    DB(1, DBFileClientConfig.class, DBFileClient.class),
    LOCAL(10, LocalFileClientConfig.class, LocalFileClient.class),
    FTP(11, FtpFileClientConfig.class, FtpFileClient.class),
    SFTP(12, SftpFileClientConfig.class, SftpFileClient.class),
    S3(20, S3FileClientConfig.class, S3FileClient.class);

    private final Integer storage;
    private final Class<? extends FileClientConfig> configClass;
    private final Class<? extends FileClient> clientClass;
}
```

### 工具类

**writeAttachment**

```java
public static void writeAttachment(HttpServletResponse response, String path, byte[] content) {
    response.setContentType(MediaType.APPLICATION_OCTET_STREAM_VALUE);
    response.setHeader("Content-Disposition", "attachment; filename=" + path);
    response.getOutputStream().write(content);
}
```
