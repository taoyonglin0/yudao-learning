## ① Why - 价值 (为什么)

### 背景与痛点

企业系统需要记录用户登录行为，用于：

1. **安全审计**：谁在什么时候登录了什么系统
2. **问题排查**：用户反馈登录异常时可以追溯
3. **合规要求**：等保要求记录登录日志
4. **风险控制**：发现异常登录行为（如频繁失败）

### 收益分析

| 收益项 | 预期效果 |
|--------|----------|
| 安全审计 | 完整的登录轨迹可查 |
| 问题排查 | 登录失败原因可追溯 |
| 风险识别 | 异常登录自动发现 |
| 合规达标 | 满足等保要求 |

### 使用场景

- 用户登录/登出记录
- 账号密码登录
- 社交登录（微信、GitHub）
- 短信验证码登录
- 异常登录检测

## ② What - 定义 (是什么)

### 核心概念

**登录日志**：记录用户登录/登出行为的日志，包含登录方式、结果、IP、设备等信息

### 一句话定义

> 登录日志是记录用户在系统中的认证行为的审计日志，包含登录类型、结果、IP、ua 等信息。

### 核心组成

```
┌─────────────────────────────────────────┐
│           登录日志体系                    │
├─────────────────────────────────────────┤
│  LoginLogDO (数据实体)                   │
│       ↓                                 │
│  LoginLogTypeEnum (日志类型)             │
│       ↓                                 │
│  LoginResultEnum (登录结果)              │
│       ↓                                 │
│  LoginLogService (日志服务)              │
└─────────────────────────────────────────┘
```

### 关键术语

| 术语 | 说明 |
|------|------|
| logType | 登录类型 (100-登录, 200-登出) |
| result | 登录结果 (0-成功, 10+-失败原因) |
| userAgent | 浏览器标识 |
| traceId | 链路追踪编号 |

## ③ How - 思维 (怎么做)

### 数据模型设计

```sql
system_login_log 表：
- id           : 日志ID
- log_type     : 日志类型 (100/101/103/104/200/202)
- trace_id     : 链路追踪编号
- user_id      : 用户ID
- user_type   : 用户类型 (1-管理员, 2-会员, 3-APP)
- username    : 用户账号
- result      : 登录结果 (0/10/20/30/31)
- user_ip     : 用户IP
- user_agent  : 浏览器UA
- created_time : 创建时间
```

### 登录类型枚举

| 类型值 | 说明 | 场景 |
|--------|------|------|
| 100 | 账号登录 | 用户名+密码 |
| 101 | 社交登录 | 微信、GitHub |
| 103 | 手机登录 | 手机号+验证码 |
| 104 | 短信登录 | 短信验证码 |
| 200 | 主动登出 | 用户自己退出 |
| 202 | 强制登出 | 管理员踢人 |

### 登录结果枚举

| 结果值 | 说明 | 场景 |
|--------|------|------|
| 0 | 登录成功 | 认证通过 |
| 10 | 账号/密码错误 | 用户名或密码错 |
| 20 | 用户被禁用 | 账号已禁用 |
| 30 | 验证码不存在 | 图片验证码过期 |
| 31 | 验证码错误 | 图片验证码错误 |

### 核心流程

```
用户登录请求
    ↓
1. 验证账号密码
    ↓
2. 创建登录日志 (logType=100)
    ↓
3. 记录登录结果 (result=0)
    ↓
4. 返回登录令牌
```

## ④ Hard - 难点 (挑战)

### 挑战 1：高并发写入

**场景**：大量用户同时登录

**解决方案**：异��写入 + 批量插入

```java
@Async
public void createLoginLog(LoginLogCreateReqDTO reqDTO) {
    // 异步写入数据库
}
```

### 挑战 2：用户账号变更

**场景**：用户修改用户名后，日志中需要保留原账号

**解决方案**：冗余存储 username

```java
private String username;  // 冗余，因为账号可以变更
```

### 挑战 3：日志量爆炸

**场景**：日均百万登录会产生大量日志

**解决方案**：日志归档 + 冷热分离

```sql
-- 定期归档（保留 90 天）
DELETE FROM system_login_log 
WHERE created_time < DATE_SUB(NOW(), INTERVAL 90 DAY);
```

## ⑥ Select - 选型 (选哪个)

### 候选方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 写数据库 | 查询方便、运维简单 | 写入压力大 | 中小型系统 |
| 写 ElasticSearch | 查询能力强 | 增加复杂度 | 大型系统 |
| 写 MongoDB | 灵活存储 | 一致性弱 | 快速开发 |
| 写文件 | 无写入成本 | 查询困难 | 归档场景 |

### 选型理由

**yudao-cloud 选择写数据库**：

1. **查询方便**：直接 SQL 查询
2. **运维简单**：一套存储
3. **性能足够**：异步写入
4. **统计方便**：直接聚合

### 核心代码

#### 1. 登录日志实体

```java
@TableName("system_login_log")
public class LoginLogDO extends BaseDO {
    private Long id;
    private Integer logType;    // 登录类型
    private String traceId;      // 链路追踪
    private Long userId;        // 用户ID
    private Integer userType;   // 用户类型
    private String username;     // 用户账号（冗余）
    private Integer result;      // 登录结果
    private String userIp;      // 用户IP
    private String userAgent;   // 浏览器UA
}
```

#### 2. 登录日志服务

```java
public interface LoginLogService {
    LoginLogDO getLoginLog(Long id);
    PageResult<LoginLogDO> getLoginLogPage(LoginLogPageReqVO pageReqVO);
    void createLoginLog(LoginLogCreateReqDTO reqDTO);
}
```

#### 3. 创建登录日志

```java
@Async
public void createLoginLog(LoginLogCreateReqDTO reqDTO) {
    LoginLogDO loginLog = LoginLogDO.builder()
        .logType(reqDTO.getLogType())
        .userId(reqDTO.getUserId())
        .userType(reqDTO.getUserType())
        .username(reqDTO.getUsername())
        .result(reqDTO.getResult())
        .userIp(reqDTO.getUserIp())
        .userAgent(reqDTO.getUserAgent())
        .build();
    loginLogMapper.insert(loginLog);
}
```

#### 4. 使用示例

```java
// 登录成功
loginLogService.createLoginLog(new LoginLogCreateReqDTO()
    .setLogType(LoginLogTypeEnum.LOGIN_USERNAME.getType())
    .setUserId(user.getId())
    .setUserType(UserTypeEnum.ADMIN.getCode())
    .setUsername(user.getUsername())
    .setResult(LoginResultEnum.SUCCESS.getResult())
    .setUserIp(ip)
    .setUserAgent(ua));

// 登录失败
loginLogService.createLoginLog(new LoginLogCreateReqDTO()
    .setLogType(LoginLogTypeEnum.LOGIN_USERNAME.getType())
    .setResult(LoginResultEnum.BAD_CREDENTIALS.getResult())
    .setUserIp(ip));
```

### 关键步骤

```

  → 校验用户名密码

Step 2: 记录日志
  → 异步写入登录日志

Step 3: 返回结果
  → 返回 Token 或错误
```
