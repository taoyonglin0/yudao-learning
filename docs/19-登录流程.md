## ① Why - 价值 (为什么)

### 背景与痛点

登录是所有业务系统的入口，是安全的第一道防线。一个健壮的登录流程需要考虑：

- **用户身份验证**：确保用户是其声称的那个人
- **会话管理**：维持用户的登录状态
- **安全防护**：防止暴力破解、撞库、Token 泄露等攻击

**没有完善的登录流程有什么问题？**

| 问题 | 场景 | 后果 |
|------|------|------|
| 密码明文存储 | 数据库被脱 | 用户密码泄露 |
| 暴力破解 | 黑客尝试大量密码 | 账号被盗 |
| Token 无过期 | Token 泄露 | 被长期冒用 |
| 无验证码 | 撞库攻击 | 批量账号被盗 |
| 无登录限制 | 同一IP大量请求 | 服务器资源耗尽 |

**完善的登录机制能带来：**

| 价值 | 说明 |
|------|------|
| ✅ 用户身份安全 | 密码加密存储、多因素认证 |
| ✅ 防止暴力破解 | 验证码、IP 限流、账号锁定 |
| ✅ 会话安全 | Token 过期、Refresh 机制 |
| ✅ 审计追溯 | 登录日志记录 |

## ② What - 定义 (是什么)

### 一句话定义

**登录流程**是用户身份验证与会话管理的完整过程，包括 credentials 校验、Token 生成、会话维护和安全防护。

### 核心组成部分

```
登录流程 = 身份认证 + 会话管理 + 安全防护 + 登录日志
```

1. **身份认证**：账号密码、短信验证码、第三方登录
2. **会话管理**：Token 生成、刷新、过期处理
3. **安全防护**：验证码、IP 限流、账号锁定
4. **登录日志**：记录登录行为，用于审计

### yudao-cloud 登录方式

| 登录方式 | 说明 |
|----------|------|
| 用户名密码登录 | 最传统的登录方式 |
| 短信验证码登录 | 手机号 + 验证码 |
| 社交快捷登录 | 微信、Gitee 等第三方登录 |
| 刷新 Token | 使用 Refresh Token 刷新访问令牌 |

### 关键术语

| 术语 | 说明 |
|------|------|
| Access Token | 访问令牌，用于接口认证 |
| Refresh Token | 刷新令牌，用于续期 Access Token |
| OAuth 2.0 | 开放授权标准 |
| Captcha | 验证码，区分人机 |
| Login Log | 登录日志 |

## ③ How - 思维 (怎么做)

### 登录流程设计

```
用户输入 → 验证码校验 → 账号密码校验 → 生成 Token → 返回登录结果
                              ↓
                      记录登录日志
```

### 核心流程详解

#### 1. 验证码校验（可选）

```java
// 验证验证码
CaptchaVO captchaVO = new CaptchaVO();
captchaVO.setCaptchaVerification(captchaVerification);
ResponseModel response = captchaService.verification(captchaVO);
if (!response.isSuccess()) {
    throw exception(AUTH_LOGIN_CAPTCHA_ERROR);
}
```

#### 2. 账号密码校验

```java
public AdminUserDO authenticate(String username, String password) {
    // 1. 查询用户
    AdminUserDO user = userService.getUserByUsername(username);
    if (user == null) {
        throw exception(AUTH_LOGIN_BAD_CREDENTIALS);
    }
    
    // 2. 校验密码（MD5加盐）
    if (!userService.isPasswordMatch(password, user.getPassword())) {
        throw exception(AUTH_LOGIN_BAD_CREDENTIALS);
    }
    
    // 3. 校验状态
    if (CommonStatusEnum.isDisable(user.getStatus())) {
        throw exception(AUTH_LOGIN_USER_DISABLED);
    }
    
    return user;
}
```

#### 3. Token 生成

```java
public AuthLoginRespVO login(AuthLoginReqVO reqVO) {
    // 1. 认证用户
    AdminUserDO user = authenticate(reqVO.getUsername(), reqVO.getPassword());
    
    // 2. 创建 Token（OAuth 2.0）
    OAuth2AccessTokenDO accessToken = oauth2TokenService.createAccessToken(
        user.getId(),
        UserTypeEnum.ADMIN.getValue(),
        reqVO.getClientId()
    );
    
    // 3. 返回登录结果
    return AuthConvert.INSTANCE.convert(accessToken);
}
```

## ④ Hard - 难点 (挑战)

### 可能遇到的问题

**难点1：密码安全**
```
场景：数据库被拖库，密码泄露
→ 解决：MD5+盐加密，强烈建议使用 BCrypt
→ 建议：定期更换密码、强制密码复杂度
```

**难点2：暴力破解**
```
场景：黑客使用密码字典暴力尝试
→ 解决：增加验证码、IP 限流、账号锁定
→ 示例：连续5次失败，锁定账号15分钟
```

**难点3：Token 泄露**
```
场景：Token 被盗用
→ 解决：Token 短期有效 + Refresh 续期
→ 建议：HTTPS 传输、Token 存储安全
```

**难点4：会话并发**
```
场景：用户异地登录，踢掉之前的会话
→ 解决：支持多设备登录或单设备强踢
→ 设计：token_id 关联，支持登出指定 Token
```

**难点5：第三方登录**
```
场景：用户通过微信登录，但未绑定账号
→ 解决：首次登录自动注册或引导绑定
→ 设计：union_id 关联同一用户
```

### 边界情况

| 边界情况 | 处理方式 |
|----------|----------|
| 账号不存在 | 返回"账号或密码错误"（防枚举） |
| 密码错误 | 记录失败次数 |
| 账号禁用 | 提示账号已禁用 |
| Token 过期 | 引导重新登录或 Refresh |
| 验证码错误 | 提示验证码错误 |

## ⑥ Select - 选型 (选哪个)

### 候选方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **Session** | 简单、服务端可控 | 不适合分布式 | 单机应用 |
| **JWT** | 无状态、易扩展 | 无法主动失效 | 微服务 |
| **OAuth 2.0** | 标准协议、功能完善 | 实现复杂 | 开放平台 |
| **SSO** | 一次登录多处使用 | 实现复杂 | 多系统 |

### 为什么选 OAuth 2.0 + JWT？

```
最后选 OAuth 2.0 + JWT，因为：
1. yudao-cloud 已实现，成熟稳定
2. 无状态，适合分布式架构
3. 支持 Token 刷新
4. 支持多种客户端
5. 社区成熟，文档丰富
```

### Token 设计

| Token | 有效期 | 用途 |
|-------|--------|------|
| Access Token | 30 分钟 | 访问接口 |
| Refresh Token | 7 天 | 续期 Access Token |

### 密码验证实现

```java
public boolean isPasswordMatch(String rawPassword, String encodedPassword) {
    // 使用 MD5 + 盐加密
    String salt = encodedPassword.substring(0, 32);
    String encoded = MD5(salt + rawPassword);
    return encoded.equals(encodedPassword.substring(32));
}
```

### 退出登录实现

```java
@Override
public void logout(String token, Integer logType) {
```

### 代码模板

```java
/**
 * 登录服务示例
 * 
 * 1. 验证码校验
 * 2. 账号密码认证
 * 3. Token 生成
 * 4. 日志记录
 */
@Service
public class AuthService {

    @Autowired
    private UserService userService;
    
    @Autowired
    private TokenService tokenService;
    
    @Autowired
    private LoginLogService loginLogService;
    
    public LoginResp login(LoginReq req) {
        // 1. 验证码校验
        verifyCaptcha(req.getCaptchaId(), req.getCaptchaCode());
        
        // 2. 认证用户
        User user = authenticate(req.getUsername(), req.getPassword());
        
        // 3. 生成 Token
        String accessToken = tokenService.createToken(user.getId());
        
        // 4. 记录日志
        loginLogService.create(user.getId(), LoginType.LOGIN, LoginResult.SUCCESS);
        
        // 5. 返回
        return new LoginResp(accessToken, user.getId());
    }
}
```
