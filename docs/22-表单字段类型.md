## ① Why - 价值 (为什么)

### 背景与痛点

在企业管理系统中，不同业务需要不同的表单：

1. **需求多样性**：员工入职、请假、报销、采购每个流程表单不同
2. **变更频繁**：业务调整需要频繁修改表单结构
3. **开发效率低**：每个表单都要前端开发，后端定义，耗时 1-2 天

### 收益分析

| 收益项 | 预期效果 |
|--------|----------|
| 开发效率 | 表单配置化，无需代码开发 |
| 业务响应 | 运营人员可直接配置表单 |
| 统一体验 | 统一的表单组件库 |
| 数据结构 | 统一的字段定义 |

### 使用场景

- BPM 流程申请表单
- 系统配置表单
- 用户信息采集
- 数据录入页面

## ② What - 定义 (是什么)

### 核心概念

**动态表单**：通过配置而非代码定义的表单，运行时动态渲染

**表单字段类型**：表单组件的种类，决定用户输入方式

### 一句话定义

> 动态表单是基于 form-generator 组件库，通过 JSON 配置实现的可视化表单构建方案。

### 表单组成结构

```
┌─────────────────────────────────────────┐
│           动态表单体系                    │
├─────────────────────────────────────────┤
│  表单配置 (conf)                        │
│       ↓                                 │
│  字段数组 (fields) ← form-generator JSON │
│       ↓                                 │
│  组件渲染 (前端 Vue 组件)                 │
└─────────────────────────────────────────┘
```

### 关键术语

| 术语 | 说明 |
|------|------|
| form-generator | 开源的可视化表单生成器 |
| vModel | 字段绑定变量名 |
| label | 字段显示标签 |
| component | 组件类型 |

## ③ How - 思维 (怎么做)

### 数据模型设计

```sql
bpm_form 表：
- id           : 表单ID
- name         : 表单名称
- status       : 状态(0/1)
- conf         : 表单配置JSON
- fields       : 字段数组(form-generator JSON)
- remark      : 备注
```

### 表单字段 JSON 结构

```json
{
  "label": "用户名",
  "vModel": "username",
  "component": "input"
}
```

### 表单字段类型 (form-generator 支持)

| 组件类型 | 说明 | 典型场景 |
|----------|------|----------|
| input | 单行输入框 | 姓名、学号 |
| textarea | 多行文本框 | 备注、描述 |
| inputNumber | 数字输入框 | 数量、金额 |
| select | 下拉选择框 | 部门、岗位 |
| multipleSelect | 多选下拉框 | 角色、标签 |
| radio | 单选按钮 | 性别、状态 |
| checkbox | 复选框 | 爱好、权限 |
| DatePicker | 日期选择器 | 出生日期 |
| TimePicker | 时间选择器 | 开始时间 |
| TimeSelect | 时间选择 | 会议时间 |
| upload | 文件上传 | 附件、图片 |
| cascader | 级联选择 | 地区、部门 |
| switch | 开关 | 是否启用 |
| slider | 滑块 | 优先级 |
| rate | 评分 | 满意度 |

## ④ Hard - 难点 (挑战)

### 挑战 1：字段验证规则

**场景**：不同字段需要不同验证规则

**解决方案**：

```json
{
  "vModel": "email",
  "rules": [
    { "required": true, "message": "请输入邮箱" },
    { "type": "email", "message": "邮箱格式错误" }
  ]
}
```

### 挑战 2：字段联动

**场景**：选择 A 字段后，B 字段选项动态变化

**解决方案**：通过 visible/disabled 控制

```json
{
  "vModel": "dept",
  "show": "position === 'employee'"
}
```

### 挑战 3：数据持久化

**场景**：表单数据需要保存到数据库

**解决方案**：结合后端实体映射

```java
BpmFormFieldRespDTO {
    label;   // 表单标题
    vModel;  // 绑定变量
}
```

## ⑥ Select - 选型 (选哪个)

### 候选方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| form-generator | 可视化、组件全 | 前端依赖 | 通用表单 |
| vue-form-maker | 轻量 | 组件少 | 简单表单 |
|formv | Vue3 支持 | 社区小 | 新项目 |
| 自研 | 完全可控 | 成本高 | 特殊需求 |

### 选型理由

**yudao-cloud 选择 form-generator**：

1. **开源活跃**：GitHub 3.5k+ star
2. **组件丰富**：内置 20+ 基础组件
3. **可视化**：拖拽式表单设计器
4. **JSON 驱动**：前后端解耦
5. **社区成熟**：遇到问题有参考

## ⑦ Impl - 实现 (细节)

### 核心实现架构

```
yudao-module-bpm/
├── dal/dataobject/
│   └── BpmFormDO.java                     # 表单实体
├── service/definition/
│   ├── BpmFormService.java               # 表单服务
│   └── BpmFormServiceImpl.java           # 服务实现
├── controller/admin/definition/vo/form/
│   ├── BpmFormSaveReqVO.java              # 保存请求
│   ├── BpmFormRespVO.java                # 响应
│   └── BpmFormFieldVO.java                # 字段定义
└── dto/
    └── BpmFormFieldRespDTO.java           # 字段DTO
```

### 核心代码

#### 1. 表单实体

```java
@TableName("bpm_form")
public class BpmFormDO extends BaseDO {
    private Long id;
    private String name;                // 表单名
    private Integer status;              // 状态
    private String conf;                 // 表单配置
    @TableField(typeHandler = JacksonTypeHandler.class)
    private List<String> fields;        // 字段数组
    private String remark;               // 备注
}
```

#### 2. 字段 DTO

```java
public class BpmFormFieldRespDTO {
    @JsonProperty("vModel")
    private String vModel;              // 绑定变量
    private String label;                 // 显示标签
}
```

#### 3. 使用示例

```java
// 后端保存表单
BpmFormSaveReqVO reqVO = new BpmFormSaveReqVO();
reqVO.setName("请假申请");
reqVO.setFields(fields);  // form-generator JSON

// 前端渲染
<form-generator :fields="formFields" />
```

### 配置项

```yaml
# 无额外配置，全由前端表单生成器配置
```

### 关键步骤

```
Step 1: 设计表单
  → 使用 form-generator 可视化设计

Step 2: 保存配置
  → fields 数组存入数据库

Step 3: 前端渲染
  → 解析 fields JSON 渲染组件

Step 4: 数据提交
  → 字段值绑定到 vModel
```
